## 为什么拆美团

美团外卖是中国互联网增长实践的集大成者。它的增长体系有几个特点值得学习：

1. **指标拆解极其彻底** — 从北极星指标到每个运营动作，都有清晰的数学关系
2. **增长引擎和产品特性高度匹配** — 补贴策略、供给侧策略、频率策略都是从产品维度推导出来的
3. **"从 0 到 1"和"从 1 到 100"的策略完全不同** — 这个差异对任何增长从业者都有参考价值

---

## 北极星指标：WUCO

美团外卖的北极星指标不是 GMV，不是订单量，而是 **WUCO（Weekly Unique Checkout Orders）— 周独立结账订单数**。

### 为什么是 WUCO 而不是其他

| 候选指标 | 为什么不选 |
|---------|----------|
| GMV | 可以被客单价拉高，不反映真实需求密度 |
| 日订单量 | 波动大，受天气/节假日影响严重 |
| 月活 | 太粗，打开 App 看看菜单不等于有需求 |
| WUCO | 周维度平滑波动，"结账"确保是真实需求，"独立"去重复计 |

### WUCO 公式拆解

```
WUCO = 周活跃用户数 × 人均周下单频次

其中：
  周活跃用户数 = 存量活跃 + 新增 - 流失
  人均周下单频次 = f(品类丰富度, 配送体验, 价格感知)
```

这个公式直接决定了两条增长路径：

| 路径 | 关注什么 | 对应策略 |
|------|---------|---------|
| 扩用户 | 周活跃用户数 ↑ | 新用户获取 + 流失用户召回 |
| 提频次 | 人均周下单频次 ↑ | 品类扩展 + 场景覆盖 + 习惯养成 |

## 品类四象限

美团做品类扩展不是随意的。每个品类按两个维度分类：

```
              高频
               |
    日常刚需    |   探索享受
   （快餐/奶茶）|  （火锅/蛋糕）
               |
 低客单价 ------+------ 高客单价
               |
    价格敏感    |   品质导向
   （特价菜）   |  （日料/西餐）
               |
              低频
```

### 不同象限的增长策略

| 象限 | 策略 | 核心动作 |
|------|------|---------|
| 高频 + 低客单 | 补贴养习惯 | 红包、满减、拼单 |
| 高频 + 高客单 | 品质保障 + 品牌合作 | 品牌馆、准时达 |
| 低频 + 低客单 | 聚合流量 | 特价频道、限时闪购 |
| 低频 + 高客单 | 场景营销 | 节日推荐、聚餐专区 |

**关键洞察**：美团先把高频低客单做到极致（快餐、奶茶），用这个品类养成用户"有事找美团"的习惯，然后用同一批用户的习惯去渗透其他象限。

## 魔法数字

美团内部有一个经典发现：

> **新用户第一周下满 3 单，次月留存率提升到 70% 以上。**

这个"3"就是魔法数字。它的含义是：用户需要在足够短的时间内体验到足够多次的完整价值交付（下单→收餐→满意），才能形成习惯。

### 魔法数字的应用

围绕魔法数字设计整个新用户体验：

1. **新手礼包设计**：不是给一张 20 元券，而是给 3 张 7 元券——逼你用 3 次
2. **首周推送策略**：前 7 天的推送频率和内容都针对"促成第 3 单"
3. **品类引导**：前 3 单推荐高满意度的品类（大品牌连锁、好评率 > 95%）

### 对其他产品的启示

每个产品都应该找自己的魔法数字：

- **你的用户需要多少次体验才能形成习惯？**
- **在多长时间窗口内完成？**
- **如何设计激励让用户达到这个数字？**

## 从 0 到 1 vs 从 1 到 100

美团的增长策略在不同阶段完全不同：

### 从 0 到 1：获客驱动

| 核心目标 | 策略 | 为什么 |
|---------|------|--------|
| 获取第一批用户 | 地推 + 补贴 | 双边市场的冷启动必须人肉推 |
| 建立供给侧 | BD 商家 | 没有商家就没有产品 |
| 验证单元经济模型 | 单城市跑通 | 先证明一个城市能赚钱 |

这个阶段不考虑效率，只考虑"能不能跑起来"。补贴不是为了赚钱，是为了让双边网络转起来。

### 从 1 到 100：留存驱动

| 核心目标 | 策略 | 为什么 |
|---------|------|--------|
| 提高留存 | 品类扩展 + 体验优化 | 用户已经来了，关键是留住 |
| 提高频次 | 场景覆盖 + 习惯养成 | WUCO 公式里频次是乘数 |
| 提高效率 | 算法调度 + 精准营销 | 从烧钱到赚钱 |

**关键转换点**：当新增用户成本开始高于留住老用户成本时，增长引擎应该从"获客"切换到"留存"。

## A/B 实验设计

美团的增长团队以实验速度著称。他们的实验设计有几个值得学习的原则：

### 实验三要素

| 要素 | 说明 |
|------|------|
| 假设 | 明确写出"我们认为 XX 会导致 YY"，不含糊 |
| 成功标准 | 上线前定好"什么算成功"，不事后挑数据 |
| 最小可行实验 | 能用 3 天验证的不用 3 周 |

### 实验分级

| 级别 | 范围 | 决策者 | 时间 |
|------|------|--------|------|
| S 级 | 全量上线 | VP | 季度级 |
| A 级 | 城市级灰度 | 总监 | 月度 |
| B 级 | 10% 用户灰度 | 经理 | 周度 |
| C 级 | 1% 用户灰度 | 个人 | 天级 |

大部分增长实验应该是 B/C 级——小范围、快速、低风险。

## 核心启示

美团的增长体系给我们三个最重要的思考：

1. **指标拆解到底。** 从 WUCO 到具体运营动作，每一步都有清晰的因果关系。你的产品有没有这样的指标拆解？

2. **策略从维度推导。** 补贴策略不是拍脑袋的，是从"高频 + 低决策成本 + 双边网络效应"这组维度推导出来的。换一组维度，补贴就可能完全无效。

3. **阶段决定打法。** 从 0 到 1 和从 1 到 100 的增长引擎完全不同。很多团队的错误是用"从 1 到 100"的精细化运营去做"从 0 到 1"的事，或者用"从 0 到 1"的烧钱打法去做已经需要效率的事。

> **增长不是一套固定打法，是一个根据产品特性和发展阶段持续调整的系统。**
